Características del Sistema

Tecnología avanzada para predicción crediticia con máxima privacidad

Privacidad Diferencial

Implementación de técnicas avanzadas de privacidad diferencial incluyendo clipping, noising y combinaciones.

Aprendizaje Federado

Entrenamiento distribuido usando Flower (FLWR) con múltiples clientes simulados y estrategias de agregación.

Múltiples Modelos

Ridge, Lasso, Random Forest y MLP (Redes Neuronales) para máxima precisión en las predicciones.

¿Cómo Funciona?

Proceso completo del sistema de aprendizaje federado

1
Preprocesamiento

División del dataset en 3 clientes simulados con codificación y normalización de características.

2
Entrenamiento Federado

Cada cliente entrena localmente y comparte parámetros con técnicas de privacidad diferencial.

3
Agregación

El servidor agrega parámetros usando FedAvg o FedMed para crear el modelo global.

4
Predicción

El modelo final realiza predicciones de score crediticio con categorización de riesgo.

Especificaciones Técnicas

  • Modelos: Ridge, Lasso, Random Forest, MLP
  • Agregación: FedAvg, FedMed
  • Privacidad: Clipping, Noising, Combinado
  • Métricas: MAE, MSE, R², Tiempo de entrenamiento
  • Framework: Flower (FLWR) para aprendizaje federado

Características del Dataset

  • Variables: Edad, Género, Educación, Ingresos
  • Objetivo: Score crediticio (300-850)
  • Clientes: 3 bancos simulados
  • Preprocesamiento: Codificación y normalización
  • División: 80% entrenamiento, 20% prueba

Cumplimiento de Requisitos del Proyecto

Modelos de Clasificación Entrenados
  • Nearest Neighbors Classifier
  • Random Forest Classifier
  • Naive Bayes Classifier
  • SVM (Support Vector Machine)
  • Neural Network MLP Classifier
  • Gaussian Process Classifier (GPC)
Métricas y Validación
  • Matriz de confusión
  • Precisión, Recall, F1-score, Accuracy
  • Comparación de desempeño entre modelos
  • Justificación del modelo óptimo
Proceso de Predicción

Los usuarios pueden subir un archivo .CSV con datos nuevos, que son procesados automáticamente (codificación y escalado) y evaluados por el modelo óptimo seleccionado. Los resultados se presentan categorizados por riesgo y pueden ser descargados en CSV.
